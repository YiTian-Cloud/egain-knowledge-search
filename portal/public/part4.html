Part 4 — Repository Structure & Integration Overview

This section is provided to explain the overall repository layout and how the different parts of the assessment are organized and integrated.

Note: No explicit coding prompt was provided for “Part 4”. I am using this section to clearly document my design decisions, repository structure, and how the different parts of the solution fit together.

Root Repository: egain-knowledge-search

The repository is structured to keep concerns cleanly separated while still allowing all parts to be demonstrated from a single entry point.

  • /docs — Part 1: High-level product and architecture documentation (system design, scalability, trade-offs, security, performance, observability)
  • /src — Part 2: Vite-based Knowledge Search UI (React + TypeScript frontend, search UX, filters, saved items, responsive design)
  • /api — Part 3: API contract design (OpenAPI spec, analytics design, rate limiting, error handling, sample HTTP clients)
  • /portal — Integration layer (Next.js) that ties all parts together into a single landing experience

Why a Next.js “Portal” Layer

To present the entire assessment cohesively, I added a lightweight Next.js layer (/portal) that serves as a unifying entry point.

The portal provides:

  • A single landing page linking all parts of the assessment
  • Navigation between documentation, Swagger UI, metrics, and frontend demo
  • Static hosting for HTML and PDF artifacts where appropriate
  • A clean separation between demo orchestration and actual implementation code

This approach avoids over-engineering while making the reviewer experience straightforward and intuitive.

Design Philosophy

  • Each part is independently understandable and reviewable
  • Frontend, API design, and architecture documentation are clearly separated
  • The portal layer focuses on presentation, not business logic
  • Static content is served statically; React is used only where needed

This mirrors how real-world teams often structure demo environments, architecture reviews, and internal platform showcases.
